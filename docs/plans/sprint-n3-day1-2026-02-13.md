# Sprint N+3 Day1 実装メモ（2026-02-13）

## 目的
- CI の品質ゲートに dependency-review を統合し、有効化時に確実にマージブロックする
- 管理画面の Supabase 読み取りを service-role 経由へ統一し、RLS 有効化に備える
- Supabase の運用テーブルに対して RLS と最低限の性能改善（FK index）を適用する

## 実装内容

### 1. CI Quality Gate 強化
- ファイル: `.github/workflows/ci.yml`
- 変更点:
  - workflow 全体の default permission を `contents: read` に固定
  - `quality-gate` に `dependency-review` を `needs` 追加
  - `quality-gate` を `always()` で実行し、各 job result を明示評価
  - `ENABLE_DEPENDENCY_REVIEW=true` かつ `DEPENDENCY_REVIEW_SUPPORTED=true` の場合のみ dependency-review 成功を必須化

### 2. Admin Server Components の service-role 統一
- 変更ファイル:
  - `src/app/admin/page.tsx`
  - `src/app/admin/approvals/page.tsx`
  - `src/app/admin/projects/page.tsx`
  - `src/app/admin/estimates/page.tsx`
  - `src/app/admin/projects/[id]/page.tsx`
- 変更内容:
  - `createServerSupabaseClient` から `createServiceRoleClient` へ置換

### 3. Supabase Day1 Security Hardening Migration
- 追加ファイル: `supabase/migrations/202602130008_day1_security_hardening.sql`

## 運用タスク（GitHub 設定）

### 必須
1. Repository variable を設定
   - `ENABLE_DEPENDENCY_REVIEW=true`
   - `DEPENDENCY_REVIEW_SUPPORTED=false`（Dependency Graph/GHAS有効化までは false）
2. Branch protection（`develop`）
   - Required status checks に `quality-gate` を設定
   - 必要に応じて `Require pull request reviews before merging` を有効化

### 推奨
1. Branch protection（`main`）にも `quality-gate` を設定
2. Dependency Graph + GHAS を有効化後、`DEPENDENCY_REVIEW_SUPPORTED=true` に変更
3. GHAS 有効時は dependency-review を required check に追加
